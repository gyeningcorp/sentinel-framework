# Pillar 1: Token Accountability
### The Audit Trail for AI Thought

---

## The Problem

Traditional logging tells you *that* an AI spoke. Token Accountability tells you *how* it thought.

A log entry that says `AI responded at 14:32` is useless when something goes wrong. You need to know:
- What information did it retrieve?
- How many reasoning steps did it take?
- Did its logic stay on topic, or did it start drifting?
- Was it making confident statements or hallucinating at high velocity?

Without this, AI governance is theater.

---

## The Concept

Every token generated by an AI agent is tied to a **Budget Identity** — a traceable record of:

- **Who** initiated the task (human or system)
- **What** the task scope was
- **How many tokens** were budgeted vs. consumed
- **What retrieval context** was used
- **Semantic density** — was the output meaningful or repetitive?

---

## Key Metrics to Track

| Metric | What It Detects |
|--------|----------------|
| Token velocity | Runaway loops, excessive reasoning |
| Semantic repetition | Hallucination spikes |
| Retrieval-to-generation ratio | Over-reliance on fabrication |
| Budget overrun % | Scope creep, prompt injection |
| Off-topic drift score | Goal misalignment |

---

## Red Flags

🚨 **Hallucination Spike** — Token velocity rises, but output confidence scores drop. The AI is generating text without grounding.

🚨 **Prompt Injection Loop** — Token budget resets unexpectedly. A malicious instruction is trying to restart the agent's context.

🚨 **Semantic Collapse** — The last 20 tokens are nearly identical to the previous 20. The agent is stuck.

---

## Implementation

### Minimum Viable (Any Stack)
1. Log all LLM calls with input/output token counts
2. Tag each call with a task ID and human initiator ID
3. Set a token budget per task — hard stop at 2x budget

### Full Implementation
1. Stream token-level telemetry to your observability stack (Datadog, Grafana, etc.)
2. Run semantic similarity checks on sliding windows of output
3. Alert on velocity anomalies (>3σ from baseline)
4. Store all token logs for minimum 90 days

### Code Example (Node.js)
```javascript
const sentinelLog = {
  taskId: uuid(),
  initiatorId: human.id,
  budgetTokens: 2000,
  model: 'gpt-4o',
  startTime: Date.now(),
  retrievalContext: ragContext.ids,
};

// After generation:
sentinelLog.tokensUsed = response.usage.total_tokens;
sentinelLog.overrun = sentinelLog.tokensUsed > sentinelLog.budgetTokens;
sentinelLog.endTime = Date.now();

await auditLog.write(sentinelLog);
```

## Executive Summary

> Your AI has an expense report. Every decision costs something. Audit it the same way you audit financial spend — with line items, budgets, and alerts when something goes over.
